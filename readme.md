# SubHunt v1.4 — Browser Replica

A mechanics-faithful browser recreation of **SubHunt v1.4**, a ~2 KB Z80 assembly game written by **Sam Heald** in 1999–2000 for the TI-83 / TI-83 Plus graphing calculators (running under the Ion shell).

**Play it here:** https://jrendel.github.io/subhunt/

> 🤖 **Proof of concept:** This entire port was produced by Anthropic's **Claude Fable 5** in a single conversation. The model was given the original `subhunt.z80` assembly source and the v1.4 readme, reverse-engineered the game's mechanics directly from the Z80 code, and produced this HTML5/canvas recreation — including the original sprite data, scoring math, collision windows, and rendering quirks.

---

## About the original

SubHunt was a popular Ion game in the TI calculator scene. You command a battleship on the surface and drop depth charges on submarines passing below — based on a Palm Pilot shooter of the same name. The original was written by Sam Heald and optimized by Wouter Demuynck (Karma), packing 15 levels, two enemy weapon types, four enemy speeds, eight depths, dual firing mechanisms, and a persistent high score table into roughly two kilobytes of machine code.

The original game is freeware and is still archived in the TI community (ticalc.org).

## What's preserved from the Z80 source

This isn't a "looks similar" remake — the game logic was translated routine-by-routine from the assembly:

- **Original sprite data, byte-for-byte** — boat, both sub orientations, depth charges, enemy shots, homing missiles, both explosion sprites
- **XOR sprite compositing** on a 96×64 1-bit buffer, just like the TI graph buffer — overlapping sprites invert pixels, exactly as on hardware
- **Point values baked into sub hulls** using the original trick: inverse text XOR'd against the sprite's hollow interior, producing white digits on the dark hull
- **Scoring formula:** sub value = `(speed + 3) × (depth + 1)` — deeper and faster subs are worth more
- **Spawn logic:** per-frame roll of `rand(255) < max(6, level)`, max 8 simultaneous subs, each at a *unique* depth (8 depth slots)
- **Enemy speed system:** speeds 0–3 gated by three cycling counters (move every 4th / 3rd / 2nd / every frame)
- **Enemy fire:** 1-in-85 chance per sub per frame; from level 8, 1-in-5 shots become **homing missiles** that steer ±1 px toward the boat
- **Exact collision windows:** 18×8 px for depth charges vs. subs; surface detonations hit when the blast lands within `boatx+1 … boatx+20`
- **The quirks:** bullets advance every other frame (`Bcounter`), 6-frame fire cooldown per side, firing only permitted on `Bcounter == 2` frames, depth charges refunded to the correct side when they leave the screen, destroyed-bullet "parked at y=70" refund behavior
- **Damage display:** the boat visibly sinks one pixel per hit — three hits and it's over, with the original 58-frame sinking animation
- **Level structure:** 15 levels, `level × 5` ships each, % destroyed tally, the 100-point *Perfect!* bonus, win screen after level 15
- **High score with initials**, defaulting to `SAM / 00100` like the original binary

The original stored its high score by writing into its own program file (self-modifying code). Here it's `localStorage` — per browser, per device, which is fittingly close to how every calculator kept its own score.

## Controls

| Action | Keyboard | Original key |
|---|---|---|
| Move | ← / → | Arrow keys |
| Fire left | Z | 2nd |
| Fire right | X | MODE |
| Pause | P | DEL |
| Quit to title | Esc | CLEAR |
| Start / continue | Enter | ENTER |

On-screen buttons are provided for touch devices.

## Running it

It's a single self-contained `index.html` — no build step, no dependencies.

- **Locally:** open `index.html` in any browser.
- **GitHub Pages:** Settings → Pages → deploy from branch → `main` / root.

## Possible extensions

- Global leaderboard (a tiny Cloudflare Worker + KV would do it — the initials entry screen is already arcade-ready)
- Adjustable tick rate to match real-hardware feel
- Sound (the original had none — the TI-83 is silent — so this would be apocryphal)
